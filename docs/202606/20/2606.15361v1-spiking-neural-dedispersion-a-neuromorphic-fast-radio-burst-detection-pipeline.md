## 摘要
在下一代射电望远镜中，实时快速射电暴（FRB）检测的成本和占地面积日益受实时去色散后端主导。我们提出了一种完整的神经形态FRB检测流水线，该流水线基于脉冲神经去色散（SND）算法构建，这是一种分层延迟相加树结构，可在脉冲编码的滤波器组数据上执行非相干去色散，支持任意试验色散量（DM）网格和可配置的分支因子。评估了三种运行模式，涵盖了从二值阈值累加到全float32精度的资源敏感度权衡。在合成北方十字滤波器组上针对Heimdall进行验证，float SND在每束244 mW功耗下达到99.3%的检测完备性，分级模式在61 mW下达到89.3%，二值模式整体达到59.4%（1.75 mW），对明亮窄事件保留91%的灵敏度。完整流水线在单个SpiNNaker 2芯片上适配所有三种模式；二值模式完全适配片内SRAM，而分级和浮点模式需要外部DRAM用于历史缓冲区。在部署的48芯片系统中，总功耗预计约为100-112 W，相较于等效GPU部署降低约10-40倍，每块板支持48个同时波束。

## Abstract
Real-time FRB detection at next-generation radio telescopes is increasingly dominated in cost and footprint by the real-time dedispersion backend. We present a complete neuromorphic FRB detection pipeline built on the Spiking Neural Dedispersion (SND) algorithm, a hierarchical delay-and-add tree that performs incoherent dedispersion on spike-encoded filterbank data, supporting arbitrary trial-DM grids and configurable branching factors. Three operating modes are evaluated, spanning a resource-sensitivity trade-off from binary thresholded accumulation to full float32 precision. Validated against Heimdall on synthetic Northern Cross filterbanks, float SND matches Heimdall at 99.3% detection completeness (244 mW per beam), graded mode achieves 89.3% at 61 mW, and binary mode reaches 59.4% overall (1.75 mW), retaining 91% sensitivity for bright, narrow events. The full pipeline fits on a single SpiNNaker 2 chip across all three modes; binary mode fits entirely within on-chip SRAM, while graded and float require external DRAM for the history buffer. On a deployed 48-chip system, total power projects to approximately 100-112 W, a reduction of order 10-40x over an equivalent GPU deployment, with 48 simultaneous beams per board.

### 论文提出的方法论

- **核心思想**：将传统的“延迟-相加”去色散算法映射到**事件驱动的脉冲神经网络（SNN）**上。由于天体物理信号（FRB）是罕见事件，大部分时间是噪声。对噪声进行阈值编码会产生非常稀疏的脉冲事件流，而SNN处理器仅在接收到输入脉冲时进行计算，从而能量消耗与输入信号的活动性成正比，而非与问题规模成正比。
- **关键技术和算法流程**：
    1.  **Spiking Neural Dedispersion (SND) 算法**：
        - **结构**：构建一个**分层延迟相加树**。输入通道被分组为簇（Cluster），在每个簇内和簇之间，用可编程的突触延迟来实现不同色散测量（DM）所需的频移。
        - **特性**：支持**任意DM试验网格**和**可配置的分支因子（B）**，解决了传统Taylor树（需2的幂次通道配对）和FDMT算法在DM网格上的限制。
    2.  **多速率扩展（Multi-rate Extension）**：
        - 为减少内存占用和功耗，将整个DM范围划分为多个波段（Bands）。每个波段根据其最低DM处的色散展宽（τsmear）**独立地降低时间分辨率**（Scrunch Factor S）。
        - 例如，高DM波段（色散展宽大）可以使用更粗糙的时间分辨率（S=16），从而显著减少历史缓冲区的深度，使二值模式的总内存（4.6 MB）能完全放在单个SpiNNaker 2芯片的片上SRAM中。
    3.  **三种运行模式**：在资源消耗和检测灵敏度之间进行权衡。
        - **浮点模式 (float, Mode C)**：使用32位浮点数进行累加，提供了最高的灵敏度（99.3%），但功耗最高（244 mW/波束）。
        - **分级模式 (graded, Mode B)**：使用8位无符号整数进行累加（允许部分和传递），是灵敏度和功耗的良好折中（89.3%，61 mW）。
        - **二值模式 (binary, Mode A)**：每个节点输出0或1，通过设置“法定阈值”（如2-out-of-4）来抑制噪声。功耗极低（1.75 mW），但灵敏度上有显著损失（59.4%）。
    4.  **流水线流程**：
        - **输入编码**：原始滤波数据经z-score归一化后，通过**delta编码器**（阈值 θ=1.5σ）生成稀疏的脉冲栅格。
        - **SND去色散**：脉冲栅格通过多速率SND树，生成DM-时间二维平面。
        - **匹配滤波**：对DM-时间平面的每一行应用DM缩放的箱车（boxcar）匹配滤波器，计算信噪比（S/N）。
        - **候选检测**：对阈值化后的S/N平面执行连通分量标记（CCL），提取候选事件。
